LAPORAN MATERI OBJECT ORIENTED PROGRAMMING (OOP) - DART

1. Pengertian OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek. Dalam OOP, program dibangun menggunakan class dan object sehingga struktur kode menjadi lebih rapi, terorganisir, dan mudah dikembangkan.

Keunggulan OOP antara lain: modularitas, reusable code, keamanan data (encapsulation), dan kemudahan maintenance pada aplikasi skala besar.

2. Class dan Object


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

Class adalah blueprint (cetakan) untuk membuat object. Object adalah hasil instansiasi dari class. Pada contoh di atas, Car memiliki atribut dan method.

3. Constructor


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

Constructor adalah method khusus yang otomatis dipanggil saat object dibuat. Constructor digunakan untuk menginisialisasi nilai atribut. Dengan constructor, data object menjadi lebih terkontrol.

4. Validasi Data (Class Mahasiswa)


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus berada dalam rentang 0.0 - 4.0');
    }
  }
}

Validasi data penting untuk menjaga konsistensi dan mencegah kesalahan input. Pada contoh ini, IPK hanya diperbolehkan dalam rentang 0.0 sampai 4.0. Jika tidak sesuai, program akan menghasilkan error.

5. Encapsulation


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

Encapsulation adalah proses menyembunyikan data internal class agar tidak dapat diakses langsung. Biasanya menggunakan private variable (diawali dengan _). Akses dilakukan melalui getter dan setter untuk menjaga keamanan data.

6. Constructor dengan Named Parameter


class Produk {
  final String id;
  final String nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

Named parameter membuat kode lebih jelas dan mudah dipahami. Parameter bisa ditulis tanpa harus mengikuti urutan tertentu. Fitur ini sangat berguna pada class dengan banyak atribut.

7. Kesimpulan

OOP dalam Dart membantu pengembang membuat program yang terstruktur, mudah dipelihara, dan aman. Konsep seperti class, object, constructor, encapsulation, dan validasi data sangat penting dalam membangun aplikasi modern. Paradigma ini sangat cocok digunakan dalam pengembangan aplikasi mobile seperti Flutter.
