1. Class dan Object

Class merupakan blueprint atau cetakan yang digunakan untuk membuat sebuah object. Dalam konsep OOP, class berfungsi sebagai rancangan awal yang mendefinisikan atribut (data) dan method (perilaku). Object adalah hasil instansiasi dari class yang dapat digunakan dalam program. Dengan menggunakan class, struktur program menjadi lebih terorganisir dan modular. 


// Nama   : M. Bagas Khoirullah
// NIM    : 251410016
// Umur   : 20 Tahun
// Jurusan: Sistem Informasi

class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

2. Constructor

Constructor adalah method khusus dalam class yang otomatis dipanggil saat object dibuat. Constructor digunakan untuk menginisialisasi nilai awal atribut agar object memiliki data yang valid sejak pertama kali dibuat. Dengan constructor, kita dapat memastikan setiap object memiliki nilai yang jelas dan terkontrol. 


// Nama   : M. Bagas Khoirullah
// NIM    : 251410016
// Umur   : 20 Tahun
// Jurusan: Sistem Informasi

class Car {
  String model;
  String color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

3. Validasi Data

Validasi data dilakukan untuk memastikan bahwa data yang dimasukkan sesuai dengan aturan yang telah ditentukan. Dalam OOP, validasi sering dilakukan di dalam constructor atau setter untuk mencegah data tidak valid masuk ke dalam object. Hal ini penting untuk menjaga konsistensi dan keamanan data dalam sistem. 


// Nama   : M. Bagas Khoirullah
// NIM    : 251410016
// Umur   : 20 Tahun
// Jurusan: Sistem Informasi

class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

4. Encapsulation

Encapsulation adalah teknik menyembunyikan data internal suatu class agar tidak dapat diakses langsung dari luar. Dalam Dart, hal ini dapat dilakukan dengan menggunakan private variable (diawali dengan underscore). Akses terhadap data dilakukan melalui getter dan setter sehingga keamanan serta kontrol terhadap data lebih terjamin. 


// Nama   : M. Bagas Khoirullah
// NIM    : 251410016
// Umur   : 20 Tahun
// Jurusan: Sistem Informasi

class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

5. Named Parameter Constructor

Named parameter constructor digunakan untuk meningkatkan keterbacaan dan fleksibilitas kode. Dengan menggunakan named parameter, pemanggilan constructor tidak harus mengikuti urutan parameter. Teknik ini sangat berguna ketika sebuah class memiliki banyak atribut agar kode tetap rapi dan mudah dipahami. 


// Nama   : M. Bagas Khoirullah
// NIM    : 251410016
// Umur   : 20 Tahun
// Jurusan: Sistem Informasi

class Produk {
  final String id;
  final String nama;
  final double harga;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
  });
}
