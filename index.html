1. Class dan Object

Tahap pertama dalam OOP adalah memahami konsep class dan object. Class merupakan blueprint atau cetakan yang digunakan untuk membuat object. Object adalah hasil instansiasi dari class yang memiliki atribut dan method.

Pada tahap ini kita mendefinisikan atribut (data) seperti model dan color, serta method (perilaku) seperti start() dan stop(). 


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

2. Constructor

Tahap kedua adalah penggunaan constructor. Constructor adalah method khusus yang otomatis dijalankan ketika object dibuat. Tujuannya untuk menginisialisasi nilai atribut.

Dengan constructor, kita dapat memastikan setiap object memiliki data awal yang jelas. 


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

3. Validasi Data

Tahap ketiga adalah validasi data. Validasi bertujuan untuk menjaga konsistensi dan mencegah kesalahan input.

Pada contoh berikut, IPK hanya diperbolehkan dalam rentang 0.0 hingga 4.0. Jika tidak sesuai, program akan menghasilkan error. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

4. Encapsulation

Tahap keempat adalah encapsulation. Encapsulation adalah teknik menyembunyikan data internal agar tidak dapat diakses langsung dari luar class.

Biasanya menggunakan private variable (diawali dengan _) dan diakses melalui getter dan setter. Hal ini meningkatkan keamanan data dalam program. 


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

5. Named Parameter Constructor

Tahap terakhir adalah penggunaan named parameter constructor. Teknik ini membuat kode lebih fleksibel dan mudah dibaca.

Named parameter memungkinkan kita memanggil parameter tanpa harus mengikuti urutan tertentu. 


class Produk {
  final String id;
  final String nama;
  final double harga;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
  });
}
